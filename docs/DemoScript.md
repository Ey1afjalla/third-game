# 演示脚本

## 演示目标

向观众展示《迷你地下城指挥官》的核心系统和技术亮点，突出战斗系统、AI 行为、Roguelike 构筑和项目完整性。

## 演示时长

总时长：5-10 分钟

## 演示流程

### 第一部分：项目介绍（1 分钟）

**演示内容**：
- 打开项目 README
- 简要介绍项目定位

**演示话术**：
> 这是《迷你地下城指挥官》，一款本地单机战术自动战斗 Roguelike 游戏。
> 
> 玩家指挥三人小队在地下城中自动战斗，通过装备、技能和策略构筑击败敌人。
> 
> 项目的特色是可解释的战斗系统——所有战斗逻辑都能通过日志和展示模式清楚追踪。

### 第二部分：核心玩法演示（3 分钟）

**演示内容**：
1. 启动游戏，展示主界面
2. 进入小队配置，展示三个角色
3. 开始一场战斗
4. 展示自动战斗过程
5. 查看战斗日志

**演示话术**：
> 首先看到主界面，我们有守卫（坦克）、游侠（输出）和术士（法师）三个角色。
> 
> 点击"开始战斗"，进入自动战斗。可以看到：
> - 角色按速度顺序行动
> - 每个行动都有明确的目标选择原因
> - 伤害计算清晰展示
> - Buff/Debuff 效果可见
> 
> 战斗结束后，可以查看详细的战斗日志，包括伤害统计、技能使用次数等。

**关键展示点**：
- 自动战斗的流畅性
- 战斗日志的完整性
- UI 的清晰度

### 第三部分：展示模式（2 分钟）

**演示内容**：
1. 开启展示模式
2. 展示角色状态机
3. 展示敌人行为树
4. 展示伤害计算公式
5. 展示技能冷却队列

**演示话术**：
> 现在打开"展示模式"，这是项目的一大特色。
> 
> 左侧展示当前所有角色的状态机状态：
> - 守卫正在 CASTING（施法中）
> - 游侠处于 IDLE（待机）
> - 敌人处于 CHOOSING_TARGET（选择目标中）
> 
> 右侧展示敌人的行为树决策：
> "骸骨队长选择'破甲斩'：目标守卫拥有最高仇恨，且护盾值大于 20"
> 
> 底部展示伤害公式：
> "基础伤害 45 - 防御减免 7 × 暴击 2.0 = 最终伤害 76"
> 
> 这让玩家能理解每一场胜负背后的系统逻辑。

**关键展示点**：
- 状态机可视化
- AI 决策透明化
- 系统的可解释性

### 第四部分：Roguelike 构筑（1.5 分钟）

**演示内容**：
1. 展示地下城路线
2. 选择路线节点
3. 战斗后的三选一奖励
4. 装备效果应用

**演示话术**：
> 这是地下城路线图，有不同类型的节点：
> - 普通战斗：获得基础奖励
> - 精英战斗：更高难度，更好奖励
> - 事件：策略选择
> - 休息：恢复生命
> - Boss：最终挑战
> 
> 战斗胜利后，会提供三选一奖励：
> - 武器：提升攻击力
> - 遗物：特殊效果（如"击杀后回复 15% 生命"）
> - 技能强化：增强已有技能
> 
> 这些选择会影响后续战斗策略。

**关键展示点**：
- 路线多样性
- 构筑深度
- 策略选择

### 第五部分：调参面板（1 分钟）

**演示内容**：
1. 打开调参面板
2. 修改角色属性
3. 调整敌人数量
4. 查看效果

**演示话术**：
> 项目内置了调参面板，方便平衡和测试。
> 
> 可以实时修改：
> - 角色基础属性
> - 敌人强度
> - 技能倍率
> - Boss 阶段阈值
> 
> 所有修改立即生效，便于快速迭代数值设计。

**关键展示点**：
- 开发工具完整性
- 数值可调性

### 第六部分：项目完整性（1.5 分钟）

**演示内容**：
1. 打开项目文档目录
2. 展示设计文档
3. 展示开发日志
4. 展示 GitHub 仓库

**演示话术**：
> 项目不仅实现了游戏功能，还有完整的文档体系：
> 
> - **需求文档**：明确项目目标和功能范围
> - **架构文档**：系统设计和模块划分
> - **战斗系统设计**：详细的战斗逻辑和公式
> - **AI 行为设计**：状态机和行为树设计
> - **数值平衡设计**：数值基线和平衡原则
> - **开发日志**：记录开发过程和决策
> - **素材生成记录**：所有美术素材的生成过程
> 
> 所有代码和文档都托管在 GitHub，有完整的版本管理和 Release。

**关键展示点**：
- 文档完整性
- 开发过程可追溯
- 项目的专业性

## 演示注意事项

### 准备工作

1. 清理浏览器缓存，确保干净演示
2. 准备一个固定随机种子，保证战斗结果可预测
3. 提前测试所有演示功能
4. 准备备用演示存档（如果演示失败）

### 演示技巧

1. **保持节奏**：每个部分控制在规定时间内
2. **突出重点**：强调可解释性、完整性和深度
3. **避免冷场**：提前准备话术，避免长时间等待
4. **互动问答**：准备常见问题的回答

### 常见问题准备

**Q: 为什么选择自动战斗？**
> A: 自动战斗让玩家专注于战前配置和路线选择，而不是重复操作。同时，通过展示模式和日志，玩家能完全理解战斗过程。

**Q: AI 如何做决策？**
> A: 使用行为树系统。每个敌人有预设的决策树，根据战场情况选择行动。所有决策原因都会记录到日志。

**Q: 数值如何平衡？**
> A: 通过调参面板快速迭代，使用固定种子验证，目标是单场战斗 8-15 回合，玩家有策略空间但不会过于简单。

**Q: 后续计划？**
> A: 添加更多角色、敌人和技能，完善美术资源，使用 Tauri 封装为桌面应用，最终在 GitHub Release 分发。
