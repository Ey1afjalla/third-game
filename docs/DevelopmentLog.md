## 2026-07-15（下午）：阶段 2 开发 - 战斗系统完整化

### 本次目标

实现阶段 2 的核心功能：
1. 完善状态机系统
2. 实现敌人行为树 AI
3. 实现 Buff/Debuff 触发逻辑
4. 优化战斗数值

### 开发内容

1. 实现状态机系统（StateMachine.ts）：
   - 完整的状态转换逻辑
   - 6种状态：idle, choosing_target, casting, recovering, stunned, dead
   - 状态转换验证
   - canAct() 检查单位是否可行动

2. 实现行为树系统（BehaviorTree.ts）：
   - 基础节点类型：Condition, Action, Sequence, Selector
   - 骸骨士兵行为树（自保、击杀、默认攻击）
   - 骸骨弓手行为树（后排优先、随机攻击）
   - 骸骨法师行为树（AOE、集火、魔法攻击）
   - 根据敌人类型获取对应行为树

3. 实现 Buff/Debuff 系统（BuffSystem.ts）：
   - addBuff() 添加效果并支持叠加
   - updateBuffDurations() 更新持续时间
   - triggerBuffEffects() 触发特定时机效果
   - 支持多种触发时机：onTurnStart, onAction, onHit, onDamaged, onTurnEnd
   - getStatModifier() 获取属性修正
   - 清除 Buff/Debuff 功能

4. 集成系统到战斗引擎：
   - 初始化每个单位的状态机
   - 执行行动时使用状态机控制状态转换
   - 敌人使用行为树进行决策
   - 回合开始时触发 Buff/Debuff 效果
   - 完整的 Buff 持续时间管理

5. 优化战斗数值平衡：
   - 骸骨士兵：80 HP → 120 HP
   - 骸骨弓手：60 HP → 90 HP
   - 骸骨法师：70 HP → 100 HP
   - 目标：延长战斗回合数到 5-10 回合

6. UI 尺寸优化：
   - 减小单位卡片尺寸
   - 调整字体大小（16px → 14px）
   - 减小间距和内边距
   - 战场高度：400px → 300px
   - 战斗日志高度：300px → 200px
   - 确保界面在标准屏幕上完整显示
