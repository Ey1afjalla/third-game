# 项目亮点介绍

## 项目概述

**项目名称**：《迷你地下城指挥官》（Mini Dungeon Commander）

**项目类型**：本地单机战术自动战斗 Roguelike 游戏

**开发周期**：2026 年 7 月开始

**技术栈**：React + TypeScript + Vite

**项目定位**：展示战斗系统设计、AI 行为实现、数值平衡和完整项目管理能力的作品集项目

## 核心亮点

### 1. 完整的战斗系统设计

**技术实现**：
- 回合制战斗引擎
- 伤害计算系统（攻击、防御、暴击、护盾）
- 技能系统（冷却、目标选择、效果触发）
- Buff/Debuff 系统（持续时间、触发时机、效果叠加）

**设计亮点**：
- 战斗逻辑与表现层分离，便于扩展和测试
- 所有数值配置化，支持热更新
- 完整的战斗日志，记录每一步操作
- 确定性随机（种子控制），战斗可复现

**展示价值**：
- 理解游戏战斗系统的核心要素
- 掌握状态管理和事件驱动架构
- 具备系统设计和模块化能力

### 2. 可解释的 AI 系统

**角色状态机**：
```
IDLE → CHOOSING_TARGET → CASTING → RECOVERING → IDLE
              ↓
           STUNNED
              ↓
            DEAD
```

每个状态转换都有明确触发条件和处理逻辑。

**敌人行为树**：
- 使用组合节点（Selector、Sequence）构建决策树
- 条件节点判断战场状态
- 行动节点执行具体操作
- 所有决策都记录原因

**示例**：
```
骸骨队长决策：
✓ 检查生命值 85/200 (42%) - 未触发狂暴
✓ 检查敌方有高防御目标
✓ 检查破甲斩冷却完成
→ 决定对守卫使用破甲斩
```

**展示价值**：
- 理解游戏 AI 的实现方式
- 掌握行为树和状态机设计模式
- 具备可调试和可扩展的系统设计能力

### 3. 深度的 Roguelike 构筑

**地下城系统**：
- 随机生成的路线图
- 多种节点类型（战斗、精英、事件、商店、休息、Boss）
- 路线选择影响游戏进程

**奖励系统**：
- 战斗后三选一奖励
- 装备（武器、护甲、饰品）
- 遗物（特殊被动效果）
- 技能强化

**构筑深度**：
- 不同装备组合产生不同战术
- 遗物效果可以叠加或协同
- 构筑选择影响后续战斗难度

**展示价值**：
- 理解 Roguelike 游戏的核心乐趣
- 掌握随机生成和奖励系统设计
- 具备游戏性设计和平衡能力

## 技术难点和解决方案

### 难点 1：战斗系统的可扩展性

**挑战**：如何设计一个既能满足当前需求，又便于后续扩展的战斗系统？

**解决方案**：
- 技能和 Buff 使用配置化设计
- 伤害计算使用公式系统，支持动态修正
- 目标选择使用规则引擎
- 所有战斗逻辑与 UI 分离

### 难点 2：AI 决策的可读性

**挑战**：如何让玩家理解 AI 为什么做出某个决策？

**解决方案**：
- 行为树每个节点记录执行结果
- 决策过程完整记录到日志
- 展示模式实时显示行为树执行路径
- 使用自然语言描述决策原因

### 难点 3：数值平衡

**挑战**：如何确保战斗既有挑战性又不会过于困难？

**解决方案**：
- 建立数值基线和伤害公式
- 使用调参面板快速迭代
- 记录每场战斗的回合数和剩余生命
- 使用固定种子验证平衡性

### 难点 4：UI 信息密度控制

**挑战**：如何在一个屏幕内展示所有必要信息，又不显得拥挤？

**解决方案**：
- 复杂信息默认折叠
- 使用标签页切换不同视图
- 展示模式可开关
- 战斗日志使用滚动区域
